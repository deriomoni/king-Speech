# King Speech — Приложение для развития ораторского мастерства

## Overview
King Speech is a mobile application designed as an educational game, inspired by Duolingo, to enhance public speaking skills. The project aims to provide an engaging and interactive platform for users to practice and improve their oratory. Key features include an AI interviewer, a "Show Time" performance simulator, and a structured progression system across various speaking exercises. The application tracks user progress, experience points (XP), and ranks to motivate continuous learning. The core vision is to make public speaking practice accessible and enjoyable through gamified learning.

## System Architecture
The application uses a hybrid architecture:
- **Frontend**: Built with Expo Router (React Native) on port 8081, providing a cross-platform mobile experience.
- **Backend**: Developed with Express and TypeScript, running on port 5000, handling API requests and business logic.
- **AI Integration**: Leverages OpenAI services through Replit AI Integrations for advanced features like speech analysis and AI interviewing.
- **Data Storage**: User progress and game state are persistently stored using AsyncStorage.

**UI/UX and Design:**
- The design theme features dark blue, gold, and green colors.
- The main screen (`app/(tabs)/index.tsx`) displays a 3D isometric staircase inspired by Omma Valley, with 6 colored modules. Steps have 3D facets, glow pulsation on active levels, and a staircase offset for modules (odd modules: left-to-right; even: right-to-left). Animations pause when the screen is not in focus.
- Adaptive width is managed using `useWindowDimensions`.
- The "Jenny" character (AI interviewer) is rendered using 7 PNG assets (331x400 RGBA) with a transparent background, featuring 8 states (idle, speaking, listening, thinking, reacting, positive, negative, outraged), blinking animations, mouth frame alternation during speech, and a breathing animation.
- Cross-platform microphone button (`MicButton.tsx`) uses Reanimated 4, `expo-linear-gradient`, Ionicons, and `expo-haptics` with three phases: idle, recording, and done.
- Show Time stage includes accent-color theming for progress bars, navigation pips, spotlights, floors, and finish overlays.
- Root layout (`app/_layout.tsx`) integrates `ThemeProvider`, `LangProvider`, and `GameProvider`.

**Technical Implementations and Features:**
- **Game System**: 67 modules, each with 6 levels (steps): warmup, interview, tonguetwister, showtime, reading, vocabulary — totaling 402 levels. Each step has a type, icon, and hints.
- **Vocabulary level**: 6th step per module. Four phases — `tutorial` (one-time `vocab_tutorial_seen` AsyncStorage flag, rules card), `spin` (single roulette block prev/current/next, `setInterval` 120ms, round 72×72 STOP button, palette #FF3B30 / #0A0F28), `play` (sub-stages `ready` with Start button + optional hint for difficulty 1, then `active` with `WaveformRecorder` + live feed + auto-finish when all synonyms found), `result` (animated counter, 2-col synonym list with `+`/`x` markers). Words from `constants/vocabularyData.ts` (300 RU words, difficulty 1/2/3 → **40/30/20s** timer). Synonym matching uses `checkSynonymAdvanced()` in `vocabularyData.ts` — 5 layers: exact, stem equality (≥4 chars), substring (≥4 chars), Levenshtein=1 on full words (≥6 chars), Levenshtein≤1 on stems. Cross-platform STT via `expo-speech-recognition` (Web Speech API on web, Apple `SFSpeechRecognizer` on iOS, Android `SpeechRecognizer` on Android) — single shared API. Started with `lang:"ru-RU"`, `interimResults:true`, `continuous:true`, `maxAlternatives:3`; listeners auto-restart on `end` while the round is active and recover from `no-speech`/`busy`. Terminal errors (`not-allowed`, `audio-capture`, `service-not-allowed`, `language-not-supported`) surface a `TextInput` fallback. Repeats are silently ignored (no red card). Native does NOT open a parallel `expo-av.Recording` (would conflict with the recognizer's audio session) — native waveform stays flat. Plugin configured in `app.json` with `microphonePermission` + `speechRecognitionPermission` (RU) and Android `androidSpeechServicePackages: ["com.google.android.googlequicksearchbox"]`. Native STT requires an Expo Launch build — Expo Go is not supported by this module. Awards `min(50, found*6)` XP via `addXP()` + `markLevelComplete()`. Session-shown IDs in AsyncStorage `@kingspeech_vocab_session_shown_v1`. Screen: `app/vocabulary-level.tsx`.
- **Progression**: Auto-scrolling to the active level on the main screen, motivational quotes, increasing content complexity, XP gain (base + bonus), and a 7-rank system (Beginner Speaker to Legend).
- **Show Time Trainer**: Two modes (Trainer and Game). Trainer mode allows theme selection without affecting main path progress. Game mode integrates with the main progression. Features unique themes per module, timed speeches (decreasing from 90s to 45s), and AI analysis (1-3 stars, XP, textual feedback).
- **Interview Mode**: Features the AI interviewer "Jenny" with a daily limit (2 interviews/day) stored in AsyncStorage. Includes reaction and response timers, a skip question option, and a final feedback screen with AI analysis. Real-time Speech-to-Text is implemented (web: SpeechRecognition API; native: waveform visualization).
- **Speech Analysis**: `services/speechAnalysis.ts` handles speech analysis with difficulty multipliers and step-type summaries.
- **Voice Recording**: Utilizes `expo-av` and web MediaRecorder for voice recording.
- **Authentication (Firebase JS SDK)**: Real auth via `lib/firebase.ts` (initializes `firebase` v12 Auth + Firestore + Storage from `EXPO_PUBLIC_FIREBASE_*` env vars; exports `firebaseConfigured` flag for graceful degradation). `context/AuthContext.tsx` exposes `signInWithGoogle()` (via `expo-auth-session/providers/google` → `GoogleAuthProvider.credential(idToken)` → `signInWithCredential`) and `signInWithApple()` (via `expo-apple-authentication` with SHA256-hashed nonce → `OAuthProvider("apple.com").credential` → `signInWithCredential`). On native, Auth persistence uses `getReactNativePersistence(AsyncStorage)`. User profile (name/gender/photoURL/onboardingCompleted) is stored in Firestore `users/{uid}` and cached in AsyncStorage `@kingspeech_user` for offline. Apple sign-in only on iOS (`Platform.OS === "ios"`, button hidden elsewhere). Email button is a stub ("coming soon" Alert). Required secrets: `EXPO_PUBLIC_FIREBASE_API_KEY`, `EXPO_PUBLIC_FIREBASE_AUTH_DOMAIN`, `EXPO_PUBLIC_FIREBASE_PROJECT_ID`, `EXPO_PUBLIC_FIREBASE_STORAGE_BUCKET`, `EXPO_PUBLIC_FIREBASE_MESSAGING_SENDER_ID`, `EXPO_PUBLIC_FIREBASE_APP_ID`, `EXPO_PUBLIC_GOOGLE_IOS_CLIENT_ID` (for native Google), optional `EXPO_PUBLIC_GOOGLE_WEB_CLIENT_ID`. `app.json` adds `ios.usesAppleSignIn: true` and the `expo-apple-authentication` plugin.
- **Avatar upload**: In `app/(tabs)/profile.tsx` the avatar is a 200×200 square (border-radius 20) above the player name. Tap → `expo-image-picker.launchImageLibraryAsync({ allowsEditing: true, aspect: [1,1], quality: 0.8 })` → fetched as Blob → `uploadBytes` to Firebase Storage `avatars/{uid}.jpg` → `getDownloadURL` written to `users/{uid}.photoURL` via `setPhotoURL()` in AuthContext. Camera edit-badge with spinner during upload. Falls back to local URI if cloud upload fails. Plain initials placeholder when no `photoURL`.
- **Cloud game progress sync**: `context/GameContext.tsx` mirrors `totalXp`, `progress`, `currentRank`, `showTimeRecordings`, `portalCompleted` to Firestore `users/{uid}/game/state` (debounced 1.5s after any local change). On `onAuthStateChanged`, hydrates from cloud once per `uid` — cloud wins only if it's strictly richer (higher XP or more completed levels). AsyncStorage stays the offline source of truth. Works only when `firebaseConfigured` is true and the user is signed in; otherwise silently no-ops.
- **Rive Animations (cross-platform)**: All `.riv` files live in `assets/rive/<name>.riv` and are imported via `require("@/assets/rive/<name>.riv")`. Metro is configured to treat `.riv` as an asset (`metro.config.js` → `resolver.assetExts` includes `"riv"`), so HMR and bundling work out of the box — no need to copy files into `public/`. Rendering goes through one shared component `components/RiveAnim.tsx`:
  - **Web** (`Platform.OS === "web"`): uses `@rive-app/react-canvas` (`useRive` hook). The asset URL is resolved first via `Image.resolveAssetSource` and, if that returns `null` (typical for non-image asset extensions on Metro web), falls back to `expo-asset`'s `Asset.fromModule(source).uri`. **Layout must be a real `new Layout({ fit: Fit.X, alignment: Alignment.Y })` instance** — passing a plain `{ fit, alignment }` object crashes inside `@rive-app/canvas` with `this.layout.copyWith is not a function`. The component handles this conversion internally via `FIT_MAP_WEB` / `ALIGN_MAP_WEB` (lowercase props → CapitalCase enum members).
  - **Native production build (Expo Launch / TestFlight)**: uses `rive-react-native` with `url={resolvedUri}`, `fit/alignment` from `FIT_MAP_NATIVE` / `ALIGN_MAP_NATIVE`.
  - **Expo Go**: renders the same `.riv` inside a `WebView` (`react-native-webview`) that loads the Rive web runtime (`@rive-app/canvas`) from unpkg CDN. The `.riv` URL is resolved via `expo-asset`'s `Asset.fromModule(...).downloadAsync()` so the WebView fetches the bundled file from Metro over the LAN. This works without a custom dev build — animations are visible directly in Expo Go on the phone. Requires Expo Go to have internet access (CDN load) and to be on the same Wi‑Fi as the Metro server. Logs WebView errors as `[RiveAnim] expo-go loadError ...`.
  - **API**: `<RiveAnim source={require("@/assets/rive/foo.riv")} style={{width:180,height:180}} fit="contain" alignment="center" autoplay />`. Optional props: `artboard`, `stateMachine`, `animation`. The component logs `[RiveAnim] web module loaded ...` and `[RiveAnim] web placeholder: useRive=... src=...` to help debug missing assets — check the browser console first if a new `.riv` shows the placeholder.
  - **Adding a new .riv**: drop it in `assets/rive/`, reference it via `require()` — that's it. Do NOT add `.riv` files under `attached_assets/` for production use; that path is not bundled into the app.
- **Web audio recording (MediaRecorder)**: `components/WaveformVoiceRecorder.tsx` web branch uses an explicit supported `mimeType` (`audio/webm;codecs=opus` → `audio/webm` → `audio/ogg;opus` → `audio/mp4`) and starts with `mr.start(250)` so chunks are flushed periodically. **Do NOT rely on `mr.onstop` to assemble the blob** — in some Chromium-in-iframe setups (Replit web preview included) `onstop` never fires after `mr.stop()`. Instead, after `mr.stop()` we call `mr.requestData()`, wait ~80ms, then build the blob directly from `session.chunks` (`new Blob(chunks, { type: mr.mimeType })`). Without this, recordings produce `chunks=N, blob=null` → empty `audioBase64` → server transcription returns `""` → speech analyzer falls back to deterministic ~2.5/10 score. Diagnostic logs are tagged `[WaveRec]` (start mimeType, fallback blob assembly, base64 length) and `[transcribe]` (server-side `inBytes`). Native (`expo-av`) path is unaffected — keep using `RecordingOptionsPresets.HIGH_QUALITY`.
- **Path background — animated blurred mesh gradient**: `components/path/MeshGradientBackground.tsx` renders 3 large circular color "blobs" (rank tint + rank accent + algorithmic complementary hue, via `useColorTween` smoothing for module changes) over a deep base (#0A0A0F dark / #F6F6FA light). Each blob animates only `translateX`/`translateY`/`scale` via Reanimated 4 shared values (`withRepeat(withTiming, -1, true)` with `Easing.inOut(Easing.sin)` and 14–25s periods + per-blob `withDelay` so motion never syncs), never `top`/`left`. Blur is applied ONCE over the blob layer — on native via `expo-blur` `BlurView` (intensity 70–80), on web via inline `style={{ filter: "blur(90px) saturate(115%)", willChange: "transform, filter" }}` (RN-Web passes through). Pauses on `!useIsFocused()` from `@react-navigation/native` (cancels all blob animations to save battery, resumes on focus). Top inner-shadow band (120px) preserved so glass header reads; soft bottom vignette (220px) adds depth. `components/path/RankBackground.tsx` now collapses all 5 rank branches into a single `<MeshGradientBackground isDark accent tintColor>` call — the old `Rank1MinimalBackground`, `RadialBloom`, `NoviceSketchPattern`, `AmateurPopArtPattern`, `ConfidentTechGrid`, `MasterArtDeco`, `ProCosmos` are kept in the file as dead code (safe to remove later). Brand purple `#9468FB` is the default Blob A color on Rank 1 / fallback.
- **Theming and Localization**: `ThemeContext` (light/dark) and `LangContext` (ru/en) provide toggleable themes and localization with persistence.
- **Game State Management**: `GameContext` manages 6 levels, XP, and ranks, persisting data in AsyncStorage (`@kingspeech_levels_v3`, `@kingspeech_xp_v3`).
- **Show Time Recordings Library**: `showTimeRecordings` in `GameContext` stores per-rank takes as `{ uri, label? }[]` (legacy `string[]` is migrated on load). The Worlds detail screen (`app/world/[id].tsx`) lets players long-press or tap the kebab on a recording to Rename, Share, or Delete it. Delete also unlinks the underlying file via `expo-file-system/legacy` on native; Share uses `expo-sharing` on native and the Web Share API (with download fallback) on web.

## External Dependencies

- **OpenAI**: Used for AI functionalities such as speech-to-text (gpt-4o-mini-transcribe) and speech evaluation (gpt-4o-mini) via Replit AI Integrations (`AI_INTEGRATIONS_OPENAI_API_KEY`, `AI_INTEGRATIONS_OPENAI_BASE_URL`).
- **AsyncStorage**: For client-side data persistence of user progress, game state, interview limits, and tutorial seen flags.
- **Expo**: Core framework for React Native development, including Expo Router for navigation and `expo-av` for audio playback/recording.
- **Web MediaRecorder API**: Used for voice recording in web environments.
- **SpeechRecognition API**: Used for real-time speech-to-text in web environments.
- **Reanimated 4**: For advanced animations, particularly in the microphone button.
- **Ionicons**: For UI icons.
- **Expo Haptics**: For haptic feedback.
- **expo-linear-gradient**: For gradient effects.

**Brand identity:** Design tokens and brand guidelines live in [`docs/brand/`](docs/brand/) (`theme/tokens.ts` is the code source of truth).
