# Промпт для Replit: Полный редизайн интерфейса King Speech

## 🎯 ЗАДАЧА

Сделай **полный визуальный редизайн** мобильного приложения King Speech, сохранив **всю существующую архитектуру, навигацию, логику и функциональность**. Не меняй ни один пользовательский флоу, ни одну фичу, ни одно состояние данных — меняй только то, **как это выглядит**.

**ВАЖНО:** Экран **Show Time** оставь без изменений — он уже выглядит так, как нужно. Все остальные экраны переделай полностью.

---

## 🎨 КОНЦЕПЦИЯ ДИЗАЙНА: "SONIC MINIMALISM"

Это собственное стилевое направление, объединяющее три эстетики:

1. **Editorial Typography** — журнальные пропорции, крупные заголовки с засечками, большие воздушные пространства, типографика как главный визуальный элемент.
2. **Neon Glassmorphism** — прозрачные стеклянные слои с лёгким размытием (backdrop-blur), фиолетовое неоновое свечение, мягкие световые сферы и градиенты, светящиеся орбитальные кольца как декорация.
3. **Brutalist Whitespace** — асимметричные композиции, неожиданные масштабы (огромный заголовок рядом с крошечной подписью), чистые геометрические формы, отсутствие лишнего декора.

### Ключевые принципы:

- **Минимализм без скуки** — каждый экран минимум элементов, но каждый элемент эстетически выверен.
- **Свет как метафора голоса** — звук в приложении визуализируется через свечение, волны, орбиты.
- **Контраст крупного и мелкого** — большие смелые заголовки + аккуратный мелкий вспомогательный текст.
- **Иерархия через размер, а не через цвет** — не перегружай интерфейс цветовыми акцентами; ведущая работа идёт через типографику и пространство.
- **Никакого скевоморфизма, никаких 3D-иконок, никаких иллюстраций "из стока"** — только чистые векторные формы, тонкие линии и свечения.

### Правила использования цвета:

- **70% поверхности** — нейтральный фон (background + surface).
- **20%** — белый/чёрный текст.
- **10% и меньше** — фиолетовые акценты и свечения.
- Жёлтый (`#FFCF34`) использовать **точечно**, не более одного раза на экран — для CTA или важного бейджа.
- Свечения (purple glow) — не более 1-2 на экран, чтобы они оставались акцентом, а не шумом.

## ✍️ ТИПОГРАФИКА

### Шрифты:

1. **Instrument Serif** (Google Fonts) — для display-заголовков и крупных акцентных текстов. Использовать `Regular` и `Italic` (курсив особенно красив и характерен — применять для эмоциональных слов в заголовках).
2. **Inter Tight** (Google Fonts) — для всего UI: основной текст, подписи, кнопки, метки, цифры. Использовать веса `400`, `500`, `600`.

### Шкала размеров:

| Стиль | Размер / Line height | Шрифт / Вес | Применение |
|---|---|---|---|
| Display XL | 56px / 60px | Instrument Serif Regular | Главные заголовки экранов (1 на экран) |
| Display L | 40px / 44px | Instrument Serif Regular | Заголовки секций |
| Display Italic | 40px / 44px | Instrument Serif Italic | Акцентные слова в заголовках |
| H1 | 28px / 34px | Inter Tight 600 | Заголовки карточек |
| H2 | 22px / 28px | Inter Tight 600 | Подзаголовки |
| Body Large | 17px / 24px | Inter Tight 400 | Основной читаемый текст |
| Body | 15px / 22px | Inter Tight 400 | Стандартный UI-текст |
| Caption | 13px / 18px | Inter Tight 500 | Метки, подписи |
| Micro | 11px / 14px | Inter Tight 500 (uppercase, letter-spacing 0.08em) | Категории, теги |

### Правила:

- На каждом экране **только один** Display XL заголовок.
- В заголовках можно микшировать стили: первое слово — Regular, второе — Italic курсивом (пример: "Your *voice*" или "Listen *closely*").
- Никогда не использовать ALL CAPS, кроме Micro-стиля.
- Letter-spacing для крупных заголовков: `-0.02em` (тайтнее).
- Никаких теней у текста.

- Лёгкое масштабирование активной иконки (1.0 → 1.1)

## 🌈 ДЕКОРАТИВНЫЕ ЭЛЕМЕНТЫ

Использовать **умеренно**, не более 1-2 на экран:

1. **Светящиеся орбитальные кольца** (как в референсе Brighter) — концентрические эллипсы из тонких линий с фиолетовым glow. Применять на онбординге, главном экране, экранах достижений.
2. **Soft purple light spheres** — большие размытые круги фиолетового света на фоне (radial gradient, blur 100px+).
3. **Тонкие звуковые волны** — синусоиды как фоновый декор на экранах с аудио.
4. **Звёздные точки/искры** (sparkle icons) — маленькие 4-конечные звёздочки рядом с AI-фичами или важными элементами (как референс smart activity planner).

**Запрещено:** 3D-рендеры, фотореалистичные иллюстрации, эмодзи в UI, стоковые иконки, любые "мультяшные" элементы.

## 🎬 АНИМАЦИИ И ПЕРЕХОДЫ

Использовать **Framer Motion** (или CSS transitions для простых случаев).

### Принципы:
- **Spring-анимации** для всех интерактивных переходов (stiffness 300, damping 30)
- **Длительность**: микро-взаимодействия 200ms, переходы экранов 400ms, fluid анимации 600ms
- **Easing**: `cubic-bezier(0.32, 0.72, 0, 1)` для большинства плавных переходов

### Обязательные анимации:

1. **Fluid tab transition** (см. выше) — жидкое перетекание индикатора в нижнем баре.
2. **Page transitions** — fade + slight slide (translateY 20px → 0).
3. **Card appearance** — fade + scale (0.96 → 1) при появлении.
4. **List items stagger** — элементы списка появляются последовательно с задержкой 50ms.
5. **Audio waveform** — пульсация столбиков при воспроизведении.
6. **Donut chart** — последовательное "вырастание" сегментов.
7. **Button press** — лёгкое сжатие (scale 0.97) при тапе.
8. **Glow pulsation** — фиолетовые свечения медленно "дышат" (opacity 0.6 ↔ 1.0, 3s loop).

## 🎁 BONUS: Финальная атмосфера

Приложение должно ощущаться как **тихий ночной разговор с самим собой** — медитативно, элегантно, технологично, но без холода. Как будто это инструмент не для "тренировки речи", а для **исследования собственного голоса**. Каждый экран — как разворот журнала премиум-класса о звуке. Свечения — это метафора того, что внутри голоса есть свет.
