Задача
В приложении King Speech уже есть экраны входа, регистрации и контекст авторизации — но они ненастоящие. Кнопки Google, Apple и телефона просто создают локального пользователя без реальной проверки. Нужно подключить настоящую авторизацию через Firebase и добавить загрузку фото аватара в профиле. Это делается в три шага строго по порядку.
Шаг 1 — Установить Firebase и настроить проект
Установить следующие пакеты: firebase, @react-native-firebase/app, @react-native-firebase/auth, @react-native-firebase/firestore, @react-native-firebase/storage, expo-auth-session, @invertase/react-native-apple-authentication. Создать файл lib/firebase.ts — в нём инициализировать Firebase приложение используя переменные окружения. Переменные брать из Replit Secrets со следующими именами: FIREBASE_API_KEY, FIREBASE_AUTH_DOMAIN, FIREBASE_PROJECT_ID, FIREBASE_STORAGE_BUCKET, FIREBASE_MESSAGING_SENDER_ID, FIREBASE_APP_ID. Экспортировать из этого файла: auth, db (Firestore), storage.
Шаг 2 — Заменить фиктивную авторизацию на настоящую
Открыть файл context/AuthContext.tsx. Сейчас функция signIn просто создаёт локального пользователя и сохраняет его в AsyncStorage — это нужно полностью заменить. Авторизация через Google: использовать expo-auth-session с Google OAuth. При нажатии кнопки Google открывается браузер Google, пользователь входит, приложение получает токен и передаёт его в Firebase через signInWithCredential. После успешного входа Firebase возвращает настоящего пользователя с реальным uid. Авторизация через Apple: использовать @invertase/react-native-apple-authentication. Работает только на iOS. При нажатии кнопки Apple открывается стандартное окно Apple ID, после входа токен передаётся в Firebase. Авторизация по номеру телефона: использовать Firebase Phone Auth. При нажатии кнопки телефона показывается поле ввода номера, Firebase отправляет SMS с кодом, пользователь вводит код и подтверждает вход. После любого успешного входа через Firebase — сохранить данные пользователя в Firestore в коллекцию users, документ с id равным Firebase uid. Сохранять: uid, displayName (имя из Google/Apple или введённое вручную), email, photoURL если есть, authMethod, createdAt, onboardingCompleted. Обновить тип UserProfile в AuthContext — добавить поле uid: string (это Firebase uid), убрать старое поле id которое генерировалось случайно. Функцию signOut заменить на Firebase signOut который разлогинивает пользователя из Firebase и очищает AsyncStorage. Слушать изменения состояния авторизации через onAuthStateChanged — если Firebase говорит что пользователь вошёл, подгружать его данные из Firestore и устанавливать в контекст. Если вышел — очищать.
Шаг 3 — Синхронизировать прогресс игры с аккаунтом
Открыть файл context/GameContext.tsx. Сейчас прогресс хранится только локально в AsyncStorage под ключами @kingspeech_xp_v4 и @kingspeech_progress_v5. Нужно добавить синхронизацию с Firestore. При каждом сохранении XP и прогресса — если пользователь авторизован, дополнительно записывать эти данные в Firestore в документ users/{uid}/gameData. Структура: xp, progress, rankIndex, updatedAt. При первой загрузке приложения — если пользователь авторизован, сначала проверить есть ли данные в Firestore. Если есть и они новее чем локальные — загрузить из Firestore. Если локальные новее — оставить локальные и загрузить их в Firestore. Это называется merge стратегия — побеждают более свежие данные. Если пользователь не авторизован — всё работает как раньше только через AsyncStorage, ничего не ломается.
Шаг 4 — Фото аватара в профиле
Открыть файл app/(tabs)/profile.tsx. Сейчас аватар это квадрат с инициалами пользователя. Нужно добавить возможность загрузить своё фото. Над именем пользователя добавить большой квадратный аватар в соотношении 1:1, минимум 200x200 пикселей с border-radius 20px. Если у пользователя нет фото — показывать инициалы как сейчас. Если фото есть — показывать его через компонент Image. На аватар наложить маленькую круглую кнопку с иконкой камеры в правом нижнем углу. При нажатии на неё открывается системный выбор фото через expo-image-picker с параметрами: только галерея, обрезка в квадрат 1:1, качество 0.8. После выбора фото — загрузить его в Firebase Storage по пути avatars/{uid}/profile.jpg, получить URL загруженного файла и сохранить его в Firestore в документ пользователя в поле photoURL. Также обновить photoURL в локальном состоянии AuthContext. Показывать индикатор загрузки пока фото загружается. Под аватаром крупно отображать имя пользователя из UserProfile.name.
Что не трогать
Всю игровую логику — уровни, задания, оценку речи, Interview с Jenny, Show Time, Словарный запас — не трогать вообще. Экраны welcome.tsx, login.tsx, profile-setup.tsx, onboarding.tsx — их дизайн не менять, только обновить логику кнопок на настоящий Firebase. Таб-бар и навигацию не трогать. Все существующие ключи AsyncStorage оставить — они нужны как локальный кеш.
Важное про Replit Secrets
Все ключи Firebase должны быть в Replit Secrets, не в коде. Перед запуском попросить пользователя добавить в Secrets: FIREBASE_API_KEY, FIREBASE_AUTH_DOMAIN, FIREBASE_PROJECT_ID, FIREBASE_STORAGE_BUCKET, FIREBASE_MESSAGING_SENDER_ID, FIREBASE_APP_ID. Без этих ключей Firebase не запустится и приложение покажет сообщение об ошибке конфигурации.
